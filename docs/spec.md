# Monzo Lattice 仕様書

格子 (lattice) 上で純正律のピッチを指定し、マルチタッチで和音を演奏する Web アプリの仕様。

各ピッチは基準ピッチ f0 に対する有理数比で表され、その比は素数の指数ベクトル
(**monzo**) で表現される。プレイヤーは二次元格子のセルや「豆」をタッチすることで
monzo を重ね合わせ、和音を作る。オクターブ (2 の累乗) は自動ボイシングで決定される。

---

## 1. 用語

| 用語 | 意味 |
| --- | --- |
| f0 | 基準ピッチ。全てのピッチは f0 に対する比で定義される |
| monzo | 素数の指数ベクトル。本アプリでは 3, p, 追加素数 (豆)、および 2 (ボイシングで自動決定) の指数の組 |
| p | ユーザーが選択する縦軸の素数 (5〜17) |
| 豆 | 3 と p 以外の素数を格子セルに追加するための UI 要素 |
| 底音 | 和音の中で一番周波数が低い音。最初にタッチされたセル/豆が担う |
| ボイシング | 各 monzo に 2 の ± 累乗をかけて実際の音高を決めること |
| バッチ期間 | タッチタイミングの揺れを吸収するために和音の変更を保留する期間 |

---

## 2. ピッチモデル

### 2.1 基準ピッチ f0

- f0 の log 周波数を、画面に表示される **log 周波数直線** (垂直直線、§5.3) 上で指定する。
- 設定画面では数値入力 (Hz) も可能にする。
- 指定可能な周波数範囲は **88 鍵ピアノと同じ**: A0 (27.5 Hz) 〜 C8 (4186.0 Hz)。
- f0 を動かしたとき、発音中のノートは **ボイシング後の monzo (2 の指数込み) が
  変わらなければピッチをグライド** させる (グライド時間は設定項目、§10)。
  monzo が変わったノートは新しく発音し直す。

### 2.2 音域 (ベース音域・中音域)

- log 周波数直線上で 2 つの音域を指定する:
  - **ベース音域**: 幅は 1 オクターブ固定。
  - **中音域**: 幅は 1 オクターブ以上 (可変)。
- 音域の実体は **f0 に対する比** として保持する。つまり f0 を動かすと音域も一緒に上下する。
- 制約:

  ```
  ベース音域_max <= 中音域_min <= 中音域_max / 2
  ```

  (右の不等式は「中音域の幅が 1 オクターブ以上」を意味する)

### 2.3 ベース機能の無効化

- オプションでベース機能を無効化できる。
- 無効化した場合、ベース音域は UI から消える。
- ボイシングへの影響は §7.3 参照。

### 2.4 素数 p

- 5 から 17 までの素数 p ∈ {5, 7, 11, 13, 17} をヘッダの select で指定する。

## 3. monzo 格子

- monzo を指定するための **二次元直交格子** を表示する。
  - 横軸 → が 3 の累乗、縦軸 ↑ が p の累乗。
  - 中央が原点 (3^0 · p^0 = 1、すなわち f0 そのもの)。
- セル (x, y) は **f0 · 3^x · p^y** の周波数のピッチに対応する
  (オクターブはボイシングで決まるため 2 の累乗は除く)。
- 格子セルは **3 cm 四方** (設定で 2〜4 cm の範囲で変更可能) で、中をタッチできる。
- 各セルにはそのセルの monzo を表示する。
- 画面上では **長辺方向に 3 の軸、短辺方向に p の軸** を割り当てる。向きは
  **3 の軸を反時計回りに 90° 回すと p の軸に一致する** ように統一する
  (横長画面: 右→ が +3、上↑ が +p。縦長画面: 下↓ が +3、右→ が +p)。

---

## 4. 豆 (追加素数)

3 と p 以外の 17 以下の奇素数 ({3, 5, 7, 11, 13, 17} ∖ {3, p}) を格子セルに追加できる。

### 4.1 表示

- 素数 1 つを **長径 1 cm の豆の形** で表現する。
- 最初は格子の外の **パレット** (ヘッダ内、§5.2) に表示する。

### 4.2 ドラッグ & ドロップ

| 操作 | 結果 |
| --- | --- |
| パレット → 格子セル | 豆をセルに **コピー** する (パレットには残る) |
| 格子セル → 格子の外 | 豆を **削除** する |
| 格子セル → 別の格子セル | 豆を **移動** する |

- ドラッグ & ドロップ中、豆が完全にセル内に入ったとき、タッチ中の全てのセルと豆を発音する。

### 4.3 セル内での整列

- ドラッグ & ドロップ中でない豆は、セル内で **左から右へ、下から上へ素数の昇順** に整列する。
- セル内のスペースがあふれた場合、**一番素数が大きい豆を消す**。
- 同じ素数の豆が重複した場合も重複を消す。

### 4.4 豆のタッチ

- 豆をタッチすると、格子セルをタッチしたときと同様に処理する (発音対象になる)。
- オクターブ (2 の累乗) を除いて:
  - セル (x, y) の **豆以外の場所** をタッチ → `3^x · p^y`
  - セル (x, y) の **豆 q** をタッチ → `3^x · p^y · q`

## 5. 画面構成

### 5.1 全体レイアウト

- **ヘッダ領域**: 画面上部。以下を同じ高さで横に並べる (§5.2)。
- 残りの領域のうち:
  - 画面が **横長なら左**、**縦長なら下** に log 周波数直線 (§5.3)。
  - アルペジオモード (§6.7) では、log 周波数直線と格子の間に
    発音コントローラ (§5.6) を置く。
  - 残りの領域は格子 (§3)。

### 5.2 ヘッダ

以下を同じ高さで並べる。幅が足りない場合は折り返し、全項目が画面内に収まるように
する (モバイル対応 §5.5):

1. **p 入力**: select。label は「p↕」。
2. **音色**: アイコンのみのボタン群。「音色」というラベルもオプションの
   テキストラベルも置かず、アイコンだけで確実に区別できるようにする
   (名前はツールチップ・読み上げ用に残してよい)。
3. **演奏モード** (§6.7): select。「直接演奏」「アルペジオ」。
4. **豆パレット**: ラベルは「つかむ」の絵文字。
5. **設定ボタン**: ⚙️。設定画面 (§5.4) を開く。

### 5.3 log 周波数直線

- log 周波数スケールの直線。次を表示・操作する:
  - f0 の指定 (§2.1)
  - ベース音域・中音域の指定 (§2.2)
  - **ボイシング結果** (現在鳴っている各音の音高) の表示 (§7)

### 5.4 設定画面

- 全ての設定項目 (§10) と **全画面トグル** を持つ。

### 5.5 モバイル対応

- iPhone, iPad, Android 端末等で使いやすいこと。
- どこをスワイプしても、画面全体がスクロールしたり更新 (pull-to-refresh) したりしないこと。
- ピンチ等でズームが変わったり、文字選択が起きたりしないこと。

### 5.6 発音コントローラ (アルペジオモード)

アルペジオモード (§6.7) のとき、log 周波数直線 (§5.3) と格子 (§3) の間に表示する。
当面は以下の 2 つのボタンを持つ。どちらも **押している間、全ノートを発音する**
(§6.7 の OR の項になる)。太鼓のように叩くとリズムを作れる。

- **リズムボタン**: 全ノートの発音のみ。
- **グライドボタン**: さらに、押している間だけ「セル移動時のノート」(§6.4) を
  グライドにする。

## 6. タッチ処理

### 6.1 基本方針

- 全て **pointer イベント** で処理し、マルチタッチを認識する。
- 最初のタッチ時には **無音を鳴らしてブラウザのミュートを解除** し、AudioContext を
  resume する (モバイルブラウザの自動再生制限対策)。

### 6.2 和音変更のバッチ化

- ここで「和音」とは **ボイシング前の和音**、すなわち **構成 monzo の集合と底音
  (§6.3) の組** を指す。どの指がどの monzo を押さえているかは含まない。
- タッチのタイミングのわずかな差を吸収するため、**和音の変更はバッチする**:
  発音中と異なる和音をもたらす最初のイベントで **バッチ期間の窓** を開き、
  窓の間は発音中の和音を維持し、窓の終わりにその時点の和音を確定する。
  窓の中のイベントは窓を延長しない。
- 窓の間に変更が打ち消されて発音中と同じ和音に戻ったら、何も確定せずに窓を
  閉じる。つまり **窓が開いている ⇔ 確定待ちの和音変更がある**。これにより、
  次の変更は常に最初から 1 バッチ期間かけて確定する。
- 指の追加・削除 (pointerdown / pointerup) もスライドによるセル移動 (§6.4) も、
  和音を変えるなら等しくバッチの対象になる。逆に和音を変えないイベント
  (発音中のセルに重ねた指の追加など) はバッチを起こさない。
- バッチ期間は 50 ms〜250 ms、10 ms 単位でオプション設定可能 (既定 100 ms)。

### 6.3 和音の構成

- 複数のセル・豆をタッチすると、monzo の **重ね合わせ** (同時発音) になる。
- **最初にタッチしたセルまたは豆が底音**、すなわち和音の一番周波数が低い音になる。

### 6.4 タッチ移動

- タッチを継続しながら隣のセルへ指を移動させたときは、それに従って和音を変更する
  (和音の変更なのでバッチ化 §6.2 の対象になる)。
- ただしミスタッチ防止のため、**セル枠の 3 % のマージンより中に入って初めて**
  移動した判定にする。
- 移動時のノートの扱いは設定 (§10) で選べる:
  - **別ノート**: 移動先のノートを新しく発音する (元のノートはリリース)。
  - **グライド**: 同じ指のノートの音高を、グライド時間 (§10) をかけて連続的に変える。

### 6.5 底音の引き継ぎ

- 底音の指を離したとき、タッチしている中で **一番最初にタッチされていたことのある指**
  が次の底音になる。
- 一度タッチが離れた指でも、もう一度タッチしたら最初のタッチ時刻が勝つ
  (タッチ開始時刻は指ごとに最初のものを記憶する)。

### 6.6 パン操作

- タッチしている指がない状態から、バッチ期間中に **1 本の指を一定以上の距離**
  (設定項目) 動かしたら、和音を作らず格子をパンする。

### 6.7 演奏モード (直接 / アルペジオ)

ヘッダの select (§5.2) で演奏モードを切り替えられる:

- **直接モード** (既定): 従来どおり、格子・豆のタッチがそのまま発音される。
- **アルペジオモード**: 格子・豆のタッチは **和音 (とボイシング) の指定のみ** を行い、
  発音は別途、すべて「押している間」の **状態の OR** で決まる:

  ```
  ノートが発音される ⇔ そのノートをタッチ中 ∨ リズムボタン押下中 ∨ グライドボタン押下中
  ```

  - log 周波数直線 (§5.3) 上のボイシング結果のノートは **タッチしている間だけ**
    発音される (アルペジオ用)。このモードでは直線上の f0・音域のドラッグ操作
    (§2.1, §2.2) は無効になる。
  - 発音コントローラ (§5.6) のボタンは押している間、全ノートを発音する。
  - 発音されていないノートは直線上に淡く表示する。
- モードを切り替えたとき、発音状態はクリアされる。

## 7. ボイシング

### 7.1 定式化

- 底音を一番低い音とし、全ての monzo に **2 の ± 累乗** をかけて音高を決める。
- ボイシングは **コスト最適化問題** に帰着する。

### 7.2 コスト関数

- コストは、**楽器のスペクトル** をもとに定義した和音の **協和感・安定感**、および
  **広がりすぎペナルティ** (係数はオプションで定義) からなる。
- スペクトルは **アタック直後** のものを使う。

### 7.3 制約

- **底音**:
  - ベース無効時: 単一のノート。中音域の制約と、**全ノートで一番下になる制約** を課す。
  - ベース有効時: **ベース音域固定のノート** と **中音域に制約されたノート** に複製する
    (2 音になる)。全ノートで一番下になる制約はベース音域のノートで自然に満たされるため不要。
- **底音以外**: 中音域固定の制約。

### 7.4 和音遷移時のオプション

和音が変わったときの挙動として、以下のいずれかをオプション画面で選択できる:

1. **つながりを考慮しない**: 前の和音と独立に、全体でボイシングし直す。
2. **同 monzo 固定**: 前の和音と同じ monzo のノートは、前と同じ音高に固定する。
3. **同指オクターブ維持**: 前の和音での指と同じ指なら、相対的なオクターブを同じにする。
   つまり、前の和音で `f0 · a_k · 2^n_k` だった音高は、次の和音で
   `f0 · b_k · 2^(n_k + m)` にボイシングする (m は k によらない整数)。

> 生む可能性がある。要検討。

### 7.5 表示

- ボイシング結果は log 周波数直線上に表示する (§5.3)。

## 8. 音響合成

- 音色として最低限以下を用意する: **正弦波、三角波、ギター、木琴**。
  - ヘッダのアイコンボタン群で切り替える (§5.2)。
- **ADSR エンベロープ** を実装する (設定画面で調整可能)。
- **部屋の反響効果 (リバーブ)** を実装する (設定画面で調整可能)。

## 12. 技術スタック (決定事項)

- 開発言語: **TypeScript**。型チェックは **`deno check`**。
- コーディングスタイル: **関数型プログラミング**。
- ビュー: **React**、状態管理: **jotai**。
- イベント処理: **RxJS** (`atomWithObservable` で jotai に接続)。
- フォーマッタ / リンタ: **`deno fmt` / `deno lint`**。
- バンドル: **`deno bundle`** で HTML エントリポイントからバンドル。
- テスト: **`Deno.test`** + **fast-check** によるプロパティテスト。
  簡潔なわりにバグ発見確率の高いテストにする。
- 自動修正・チェックは **`deno task`** で一括実行できるようにする
  (`deno task fix` / `deno task check`)。
- CI: **GitHub Actions**。デプロイ先: **GitHub Pages**。

開発環境の詳細は [README.md](../README.md) を参照。
